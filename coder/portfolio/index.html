Zanies - Case Study

Project Overview

Zanies - Case Study is my UX/UI project. With this project I wanted to improve the way that a client buys a ticket online for a certain show time at Zanies.

 

The Problem

"I am a 43 year old male, fun guy. I am trying to get tickets for a show at Zanies but when I pick the show I want to get tickets for, I can't because after I click the show, they (Zanies website) tell me that tickets for that specific show are sold out, which makes me feel very frustrated (angry even), that I want pUncH somone in the face (Zanies webmaster)."

 

 

So...

I was tasked with designing a better way to get tickets online at Zanies portal.

 

Research

I began by framing the problem.

How might we...

  • Create: a portal that has a simple way to purchase a ticket for a show time
  • For: frustrated clients
  • To: check all the show times of a specific artist on an given day, and the tickets available for that specific show time
  • So that: clients could get (purchase) the tickets they wanted easily and fast

 

 

Documentation

Understanding the problem, I developed documentation to direct and support my design. I created:

  • User stories, outlining tasks a user should complete.
  • A page flow, illustrating the users path through the application.

With well defined user stories, a solid page flow, and detailed information about each page, I started prototyping.

 

 

Prototyping

I began, as I do every project, with pen and paper. I find that writing things out, and letting the pen draw, is one of the most effective ways to move past bad designs and refine toward good designs.

This project required:

  • All prototyping beyond pen and paper was done using JustInMind.
  • The prototype (final website design) was developed with HTML / CSS / jQuery / Bootstrap.

 

 

Iteration and Testing

Iteration and testing was done internally only.

  • I presented the prototype to the class.
  • Then they gave me feedback and some recommendations about it.
  • Through this testing I was able to refine and streamline the prototype further.
  • With a design the whole class felt comfortable with, I then proceded coding the final design.

 

Final Design

There was a lot work to do to bring it to a final design state:

  • Hours of code were needed to put all the pieces together on the web.
  • This project was completed on time and the proposed website worked as expected.
  • And I still need to decide if I will present this to the Zanies stakeholders to see if they are able to find this proposal valuable.
